Problem Statement

    

Cat loves making trees. Cat's friend Dog will have a birthday soon, so Cat has decided to make a special tree for Dog.



Cat likes the numbers 4 and 7. Therefore, the weight of each edge in the tree will be either 4 or 7.



Dog likes balance. A simple non-empty path in the tree is balanced if it contains the same number of weight-4 and weight-7 edges. (Note that an empty path is not balanced.) Direction does not matter: a path from X to Y is the same as a path from Y to X.



Dog's favorite number is favorite. Cat would like to encode this number into the tree as the number of balanced paths it contains.



You are given the int favorite. Find a tree that contains at most 51 vertices and has all the above properties. Return a vector <int> containing a description of the tree in the format specified below. If there are many such trees, you may return any of them. If there are no such trees, return an empty vector <int> instead.



Suppose that you want to return a tree with V vertices. (Note that V must be between 1 and 51, inclusive.) First, label those vertices 0 through V-1. Then, return a vector <int> X with exactly 3V-2 elements. X[0] should be the number of vertices. The rest of X should be the list of edges. More precisely: For each edge in your tree there must be an i between 0 and V-2, inclusive, such that X[3*i+1] and X[3*i+2] are the vertices connected by that edge and X[3*i+3] is the weight of the edge: either 4 or 7.
